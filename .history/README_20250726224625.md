# Spring Boot gRPC REST Demo

A comprehensive Spring Boot application demonstrating both REST API and gRPC services with enhanced security features.

## Features

- **REST API**: Full CRUD operations for Users and Products
- **gRPC Services**: High-performance communication
- **JWT Authentication**: Secure token-based authentication
- **Rate Limiting**: Protection against abuse
- **Comprehensive Logging**: Detailed request/response logging
- **Swagger Documentation**: API documentation
- **H2 Database**: In-memory database for development

## Security Features (OWASP Top 10 Protection)

This application implements comprehensive security measures to protect against OWASP Top 10 threats:

### 1. Broken Access Control
- Role-based access control (USER, ADMIN)
- Method-level security with `@PreAuthorize`
- URL-level security configuration

### 2. Cryptographic Failures
- BCrypt password hashing
- JWT tokens signed with HMAC-SHA256
- Configurable JWT secret (minimum 32 characters)

### 3. Injection
- Parameterized queries via Spring Data JPA
- Input validation with Bean Validation

### 4. Insecure Design
- Secure authentication patterns
- Proper error handling

### 5. Security Misconfiguration
- Secure HTTP headers (CSP, HSTS, X-Frame-Options)
- Restricted CORS origins
- Protected H2 console and Swagger UI

### 6. Vulnerable Components
- Regular dependency updates recommended
- Use OWASP Dependency-Check

### 7. Authentication Failures
- Strong password policy (min 8 chars, complexity requirements)
- Account lockout after 5 failed attempts (15-minute lockout)
- JWT token expiration

### 8. Software Integrity
- Input validation and sanitization

### 9. Security Logging
- Comprehensive audit logging
- Sensitive data masking in logs

### 10. Server-Side Request Forgery (SSRF)
- **URL Validation**: Comprehensive SSRF pattern detection
- **Protected HTTP Service**: Safe external request methods
- **Network Restrictions**: Configurable allowed hosts and schemes
- **Request Filtering**: SSRF protection filter for incoming requests

### CSRF Protection
- **Status**: Disabled (Acceptable for REST APIs with JWT)
- **Reason**: JWT tokens provide sufficient protection against CSRF attacks
- **Note**: Enable CSRF protection for web applications with session-based authentication

## Configuration

### Environment Variables

Set these environment variables for production:

```bash
# JWT Configuration
JWT_SECRET=your-super-secret-jwt-key-must-be-at-least-32-characters-long

# CORS Configuration
CORS_ALLOWED_ORIGINS=https://yourdomain.com,https://api.yourdomain.com

# Security Configuration
SECURITY_RATE_LIMIT_REQUESTS_PER_MINUTE=10
SECURITY_ACCOUNT_LOCKOUT_MAX_ATTEMPTS=5
SECURITY_ACCOUNT_LOCKOUT_LOCK_DURATION_MINUTES=15
SECURITY_PASSWORD_MIN_LENGTH=8
```

### Application Properties

Key security configurations in `application.yml`:

```yaml
jwt:
  secret: ${JWT_SECRET:your-super-secret-jwt-key-must-be-at-least-32-characters-long-for-production}
  expiration: 86400000 # 24 hours

security:
  cors:
    allowed-origins: ${CORS_ALLOWED_ORIGINS:https://yourdomain.com}
  rate-limit:
    requests-per-minute: 10
  account-lockout:
    max-attempts: 5
    lock-duration-minutes: 15
  password:
    min-length: 8
```

## Running the Application

### Prerequisites
- Java 17+
- Maven 3.6+

### Development
```bash
mvn spring-boot:run
```

### Production
```bash
# Set environment variables first
export JWT_SECRET="your-production-secret-key"
export CORS_ALLOWED_ORIGINS="https://yourdomain.com"

# Run the application
mvn spring-boot:run
```

## API Endpoints

### Authentication
- `POST /auth/register` - Register new user
- `POST /auth/login` - Login and get JWT token
- `GET /auth/validate` - Validate JWT token

### Users (Admin only)
- `GET /api/users` - Get all users
- `GET /api/users/{id}` - Get user by ID
- `POST /api/users` - Create user
- `PUT /api/users/{id}` - Update user
- `DELETE /api/users/{id}` - Delete user

### Products (User/Admin)
- `GET /api/products` - Get all products
- `GET /api/products/{id}` - Get product by ID
- `POST /api/products` - Create product
- `PUT /api/products/{id}` - Update product
- `DELETE /api/products/{id}` - Delete product

### Development Tools (Admin only)
- `GET /h2-console` - H2 Database Console
- `GET /swagger-ui.html` - Swagger API Documentation

## Security Best Practices

1. **Change Default Passwords**: Update default user passwords in production
2. **Use HTTPS**: Always use HTTPS in production
3. **Regular Updates**: Keep dependencies updated
4. **Monitor Logs**: Monitor application logs for suspicious activity
5. **Backup Data**: Regular database backups
6. **Environment Separation**: Use different configurations for dev/staging/prod

## Testing Security Features

### Test Account Lockout
```bash
# Try to login with wrong password 5 times
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"john@example.com","password":"wrongpassword"}'
```

### Test Rate Limiting
```bash
# Make multiple requests quickly
for i in {1..15}; do
  curl http://localhost:8080/api/test/rate-limit
done
```

### Test Password Policy
```bash
# Try to register with weak password
curl -X POST http://localhost:8080/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"test@example.com","password":"weak"}'
```
